# 代码语言与注释规则

Codex 修改本仓库代码时，必须遵守以下规则。

## 输出语言

- 注释、错误信息、日志、命令行提示、UI 文案、占位文案和测试输出默认使用中文。
- 第三方 API 名称、技术名词、品牌名、命令参数、环境变量、协议字段、文件路径和代码标识符可以保留英文。
- 包装第三方英文错误时，外层必须补充中文上下文。

## 注释规则

- 导出的函数、复杂业务函数、跨模块入口必须写函数级注释，说明用途、关键参数、返回值和重要副作用。
- 导出的 `type`、`interface`、`class`、`enum` 必须写类型级注释。
- 导出类型的每个字段都必须写字段注释，说明含义、单位、是否可选、默认行为或约束。
- 模块级常量、配置项、状态变量、缓存变量和含义不直观的局部变量必须写注释。
- 复杂分支、重试、降级、权限判断、路径校验、事件顺序转换等逻辑，应在关键代码块前写简短注释。
- 不要给自解释的临时变量、简单回调或直观工具函数添加空泛注释。

## 注释质量

- 注释应解释“为什么”和“这个概念是什么”。
- 避免复述代码已经清楚表达的动作。
- 好的注释示例：`缓存本轮 run id，保证后续事件可以被前端归并到同一次运行。`
- 差的注释示例：`把值赋给变量。`

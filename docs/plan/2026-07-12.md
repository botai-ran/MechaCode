## 2026-07-06：确定方向与事件协议草案

目标：把本周开发范围定下来，先明确 Agent 运行时和桌面端之间要传递什么事件。

- [x] 完成本周开发计划文档。

- [x] 设计 `AgentRunEvent` 事件协议草案。
- [x] 明确事件流至少需要覆盖模型请求中、模型输出、工具调用、工具结果、错误和完成状态。
- [x] 确认 Tauri 到前端的事件传递方案，优先使用 Tauri event emit。

事件协议草案：

```ts
type AgentRunEvent =
  | { type: "run_start"; runId: string }
  | { type: "model_request_start"; runId: string }
  | { type: "message_start"; messageId: string; role: "assistant" }
  | { type: "text_delta"; messageId: string; text: string }
  | { type: "message_done"; messageId: string }
  | { type: "tool_call_start"; toolCallId: string; name: string; input: unknown }
  | { type: "tool_call_done"; toolCallId: string }
  | { type: "tool_result"; toolCallId: string; output: unknown }
  | { type: "error"; message: string }
  | { type: "run_done"; runId: string };
```

事件含义：

- [x] `run_start`：一次 Agent run 开始。
- [x] `model_request_start`：开始请求模型接口，桌面端显示“思考中”。
- [x] `message_start`：开始生成一条 assistant 消息，前端创建空消息占位。
- [x] `text_delta`：模型返回一段增量文本，前端追加到对应 assistant 消息。
- [x] `message_done`：当前 assistant 消息生成完成。
- [x] `tool_call_start`：模型请求调用工具。
- [x] `tool_call_done`：工具调用动作结束。
- [x] `tool_result`：工具执行结果回传。
- [x] `error`：运行过程中出现错误。
- [x] `run_done`：一次 Agent run 完成。

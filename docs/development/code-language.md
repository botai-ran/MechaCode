# 代码语言约定

## 中文输出

项目中的注释、报错信息、日志打印、命令行提示、界面提示、占位文案和测试脚本输出默认使用中文。

允许保留英文的内容仅限于：

- 代码标识符，例如变量名、函数名、类型名、类名和事件名。
- 技术名词和品牌名，例如 React、Tauri、OpenAI、DeepSeek、CLI。
- 命令、参数、环境变量、协议字段和文件路径，例如 `--provider`、`OPENAI_API_KEY`。
- 第三方库要求固定使用的字符串字面量。

新增代码时，如果错误会展示给用户、写入终端、进入日志或出现在 UI 中，应优先写成中文。需要包装第三方英文错误时，应在外层补充中文上下文。
